# Git Workflow для проекта SFMShop

## Использованные команды Git
- git init - инициализация репозитория
- git add - добавление файлов в staging area
- git commit -m <Краткое описание коммита> - указание информации об изменениях
- git push origin <название ветки> - отправление коммитов в удаленные репозиторий
- git pull origin <название ветки> - получение изменений из удаленного репозитория
- git checkout - переключение между ветками
- git branch - просмотр существующих веток
- git show HEAD - просмотр изменений в последнем коммите

## Созданные ветки
- main - основная ветка проекта
- feature/add-inventory-management - разработка системы склада
- feature/add-shipping - разарботка системы доставки

## Разрешенные конфликты
- Конфликт в src/models/product.py при слиянии feature/add-inventory-management
- Конфликт в src/models/product.py при слиянии feature/add-shipping

## Стратегия работы с ветками
1) Создаем отдельную ветку под конкретную бизнес-задачу
2) После завершения задачи добавляем файлы в staging area
3) Пишем осмысленный commit
4) Делаем git pull из основной ветки
5) Делаем git push
6) Создаем Pull Request
7) При возникновении конфликтов либо разрешаем их, либо переписываем код чтобы соблюдал условия всех изменений
