Диагностика проблем с загрузкой сайта
DNS-запрос:
Преобразовать доменное имя сайта в IP-адрес можно с помощью нескольких инструментов:
Воспользоваться онлайн-сервисом 2ip.ru;
Команда nslookup в командной строке Windows (в Linux требуется доп установка)

Возможные проблемы:
Отсутствуют или неправильно настроены нужные DNS-записи на стороне хоста;
Сервер не работает;
Не удалось найти ip по домену - Non-existent domain

На начальном этапе подключения домена следует правильно настроить DNS-записи,
либо проверить правильность настройки DNS-записей.
Если записи настроены, но DNS-запрос не нашел результата, проверить 
правописание домена либо перезагрузить сервер

2. TCP-соединение
Простой способ проверить TCP-соединение можно с помощью следующих внутренних инструментов ОС:
Команда ping. Команда ping обменивается пакетами с сервером (автоматически определяет его IP),
считает максимальное, минимальное и среднее время, а также процент потерь. В Windows производит
всего 4 обмена, тогда как в Linux выполняет обмен пока не остановишь вручную
2. На Linux также можно воспользоваться внутренней командой "nc -zv <domen> <port>".
С ее помощью можно проверить соединение с различными портами, что помогает детально разобраться
с причинами.

Возможные проблемы:
Firewall блокирует порты;
Процесс на одном из портов не запущен;
Большая задержка или большой процент потерь пакетов.

Проверить какой из портов не запущен можно с помощью раннее описанной командой - "nc -zv <domen> <port>".
Это даст понимание какой из процессов не запущен или с каким из процессов возникают трудности.
Следует попробовать перезапустить сетевой сервис или сменить сеть..
Проблемы с Firewall решаются либо на стороне хостинг-провайдера, либо изнурительным трудом с помощью множества команд.


3. HTTP-запрос

Проверить работу HTTP-запросов можно с помощью следующих инструментов:
Postman - специальное приложение, имеет широкий набор инструментов для проверки HTTP-запросов;
С помощью инструмента разработчика браузера DevTools;
С помощью команды curl в консоли.

Возможные проблемы:
Сервер возвращает ошибку (ошибку клиента, сервера)
Таймаут соединения;
Некорректные заголовки.

Для решения подобных проблем следует проверить корректность заголовков,
проверить логи сервера. Он отобразит причины ошибки, если это ошибки сервера.
Следует проверить корректность отправляемых данных.
В случае возникновения таймаута, следует увеличить его время в конфигурации, например, nginx.

4. Обработка на сервере

Проверить работу этапа можно следующими методами:
Анализировать логи сервера;
С помощью команды top;
Проверять время выполнения этапов кода с помощью datetime.now()

Возможные проблемы:
Неэффективный код;
Нехватка ресурсов сервера (к примеру uvicorn может использовать 99% RAM или CPU и сервер остановит запрос);
Неэффективные запросы в БД;
Необработанные ошибки.

Логи сервера покажут, где возникла ошибка, а также при использовании datetime.now() покажут неэффективные участки кода.
В данном случае следует оптимизировать код или запросы в БД, чаще всего это сильно ускоряет работу программы.
Следует обрабатывать вероятные ошибки, чтобы работа программы не останавливалась.
Если во время выполнения запроса команда top показывает высокую нагрузку на ресурсы сервера, следует
увеличить мощность процессора или объем оперативной памяти

5. HTTP-ответ

Проверить работу этапа можно с помощью инструенмта разработка DevTools.
Он детально показывает время получение ответа, ошибки и проблемы при некорректном HTTP-ответе.

Возможные проблемы:
Тяжелые статические файлы;
Слишком большой ответ;
Неверный тип данных.

Для решения подобных проблем следует сжимать тяжелые файлы, уменьшать размер ответа,
указывать верный тип данных.

6. Рендеринг в браузере

Для проверки данного этапа можно использовать DevTools.
Покажет ошибки при обработке полученных данных, ошибки отображения на странице.

Возможные проблемы:
Слишком много запросов;
Долгая обработка данных;
Большой размер статических файлов.

Для решения подобных проблем стоит начать с оптимизации обработки данных, изображений.
Воспользоваться инструментами, помогающими обработать большое количество запросов.
